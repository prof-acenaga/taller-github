Guía rápida de Git & GitHub

Git es un sistema de control de versiones distribuido que permite trabajar de manera colaborativa y segura. GitHub amplía sus posibilidades al ofrecer alojamiento en la nube, trabajo en equipo y herramientas visuales. A continuación encontrarás los comandos más importantes, con explicaciones prácticas pensadas para estudiantes.

Instalar Git

GitHub ofrece versiones para todas las plataformas principales. Instálalo desde:

  • Windows: windows.github.com
  • Mac: mac.github.com
  • Linux / POSIX: git-scm.com

Puedes usar GitHub Desktop para una interfaz gráfica o la terminal para dominar los comandos manuales.

Configurar Herramientas

Antes de empezar, configura tu identidad de usuario:

  • $ git config --global user.name "[nombre]" — Define tu nombre de autor para todos los commits.
  • $ git config --global user.email "[correo]" — Asocia tu dirección de correo con tus confirmaciones.
  • $ git config --global color.ui auto — Activa colores en la salida de la terminal.

Crear Repositorios

Puedes iniciar un nuevo proyecto o clonar uno existente desde la web.

  • $ git init [project-name] — Crea un repositorio vacío en tu equipo.
  • $ git clone [url] — Descarga un proyecto remoto completo (código + historial).

Efectuar Cambios

Flujo básico de trabajo: revisar, preparar y confirmar.

  • $ git status — Muestra qué archivos se han modificado o están pendientes de subir.
  • $ git diff — Compara los cambios realizados antes del commit.
  • $ git add [file] — Agrega un archivo al área de preparación.
  • $ git diff --staged — Muestra diferencias entre lo preparado y la última versión.
  • $ git reset [file] — Quita un archivo del área de preparación sin borrar su contenido.
  • $ git commit -m "[mensaje]" — Registra los cambios con un mensaje descriptivo.

Cambios Grupales

Las ramas permiten desarrollar funciones o ideas sin alterar la versión principal.

  • $ git branch — Lista todas las ramas del repositorio.
  • $ git branch [nombre] — Crea una nueva rama.
  • $ git checkout [rama] — Cambia a la rama especificada.
  • $ git merge [rama] — Combina la rama con la actual.
  • $ git branch -d [rama] — Elimina una rama local.

Nombres del Archivo de Refactorización

  • $ git rm [file] — Borra el archivo del proyecto y lo marca para commit.
  • $ git rm --cached [file] — Elimina el seguimiento de un archivo sin borrarlo localmente.
  • $ git mv [file-original] [file-renamed] — Cambia el nombre de un archivo y prepara el cambio para commit.

Suprimir Tracking

Excluye archivos temporales o irrelevantes con un archivo especial llamado .gitignore.

*.log
build/
temp-*

Estos patrones indican qué archivos o carpetas ignorar.

$ git ls-files --other --ignored --exclude-standard — Muestra los archivos ignorados del proyecto.

Rehacer Commits

  • $ git reset [commit] — Revierte commits, conservando los cambios locales.
  • $ git reset --hard [commit] — Borra el historial y restaura el estado exacto del commit indicado.

Repasar Historial

  • $ git log — Muestra el historial de commits en la rama actual.
  • $ git log --follow [file] — Muestra los cambios de un archivo específico (incluso renombrado).
  • $ git diff [rama1]...[rama2] — Compara el contenido entre dos ramas.
  • $ git show [commit] — Detalla los cambios y metadatos de un commit concreto.

Guardar Fragmentos

  • $ git stash — Guarda temporalmente tus cambios sin hacer commit.
  • $ git stash list — Lista los cambios almacenados.
  • $ git stash pop — Restaura el último stash.
  • $ git stash drop — Elimina el stash más reciente.

Sincronizar Cambios

  • $ git fetch [bookmark] — Descarga el historial remoto sin aplicarlo aún.
  • $ git merge [bookmark]/[rama] — Integra los cambios descargados.
  • $ git push [alias] [rama] — Envía tus commits a GitHub.
  • $ git pull — Descarga y fusiona los cambios remotos.
